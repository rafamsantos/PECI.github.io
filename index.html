DETI Access Control System

ABOUT THE PROJECT

O domínio da tecnologia encontra-se em constante desenvolvimento e sistemas de controlo de acessos não estão excluídos. Existe, portanto, uma maior necessidade de substituir sistemas desatualizados devido a crescentes fatores de inconveniência, quer estes sejam de segurança ou até mesmo mera facilidade de utilização.

Foi projetado um sistema que utiliza tecnologia Near Field Communications(NFC) e o telemóvel pessoal para controlar acessos às salas do Departamento de Eletrónica Telecomunicações e Informática(DETI).

Para executar este projeto é necessário que todas as portas com acesso restrito estejam equipadas com um módulo NFC e que os servidores da Universidade de Aveiro estejam ativos e aptos a reconhecer todo o pessoal docente, não docente e alunos através de informação trocada entre a app e o módulo NFC previamente instalado na porta. Este projeto visa desenvolver uma aplicação móvel para que seja possível a entrada em todas as salas do departamento às quais o utilizador tem acesso. Este método tem a vantagem de ser mais prático e user-friendly, podendo ser aplicada neste e noutros departamentos futuramente, bastando apenas a instalação da app no seu smartphone.

Aqui, vamos descrever a pesquisa de soluções para oferecer uma alternativa ao uso de cartões de acesso que são atualmente utilizados e se revelam uma solução falível devido à sua transmissibilidade e possibilidade de perda.

A solução passa por 2 métodos distintos baseados em NFC e Wifi.

Arquitetura

O sistema geral é constituído por 3 componentes principais: O sistema de abertura de portas, o servidor e a aplicação. O Sistema de abertura de porta é constituído pelo microcontrolador instalado na porta, o servidor possui uma base de dados incluída e a aplicação móvel inclui suporte de Flask.

Existem dois casos de uso principais, NFC e Wifi.

  • NFC

    O utilizador deve, com a aplicação já aberta no seu smartphone, pressionar um botão para pedir ao servidor para gerar um código NFC que será enviado para o seu dispositivo e simultaneamente armazenado numa base de dados associada ao Media Access Control(MAC) do seu dispositivo. De seguida deverá aproximar o smartphone do leitor NFC da porta por onde deseja passar, para que lhe seja concedido acesso. Neste processo, o leitor NFC fará um pedido de verificação ao servidor para conferir as suas credenciais e respetivas permissões.

  • Wifi

    O concedente, alguém com permissões elevadas(ex. Segurança do departamento), faz o login na aplicação através da qual terá acesso remoto a todas as portas com acesso restrito podendo abrir qualquer uma sem estar no local.

  • Aplicação Móvel

    Foi desenvolvida para permitir ao utilizador pedir acesso a uma porta, ver o seu histórico de acessos e gerir as suas credenciais, tudo à distância de um clique no seu smartphone pessoal. A aplicação foi desenvolvida usando React Native. Como framework foi utilizado o Flask com o objetivo de ser um rest api para realizar o Routing, ou seja, para transferencia de informação entre o backend e o frontend.

  • Sistema de Autenticação

    A implementação do sistema de autenticação de 2 fatores com a utilização do email da UA começa a nível do frontend nos screens escritos com javascript. Nestes, o input do utilizador é lido e utilizado pelo backend para analisar as credenciais da conta do utilizador bem como o código de verificação.

    No caso de criar uma nova conta, os dados são guardados na base de dados, se satisfizerem as condições de segurança. Em caso contrário, serve para comparar as credenciais com os dados da base de dados para decidir se procede ou não com o início da sessão do utilizador. Adicionalmente, usa-se o sendgrid_api, fornecido pela twillio, para realizar o envio dos emails de forma segura. Os códigos enviados tanto no registo como no login são válidos durante 5 minutos. Já o token de sessão criado mal ocorre login encontra-se válido durante 24h.

    As condições de segurança relativas ao registo de um novo utilizador ocorrem tanto a nível do email como da palavra passe. O email inserido tem de terminar em @ua.pt e a palavra passe tem uma extensão de 12 a 128 caracteres. Utilizando a função zxcvbn da biblioteca do mesmo nome, a força da palavra passe é quantificada, o que permite forçar uma palavra passe mais resistente a ser inserida. Verificar se a palavra passe comparece numa lista de credenciais extraídas em falhas de segurança, ocorre com o api fornecido pelo site Have I Been Pwned. A nossa implementação do sistema de autenticação padece de decorrer no local incorreto.
